What is CSS?

CSS, standing for Cascading Style Sheets, is generally used in combination with HTML and Javascript for Web Development. These three each serve a different purpose towards making a proper web page; HTML gives the structure, CSS gives the visuals, and Javascript gives functionalities.

CSS provides a variety of properties that could be used to customize each and every tag that you have in your HTML code. It follows a property:value format to change visuals. Ranging from simple fonts(family, size, weight), entity alignments(for text, border, div, etc), and color specifications (for background, text, border, etc) to more complex transforms (scaling, rotating, etc), transitions, and much, much more.

All in all, its all you need to give a brand new look to your currently bland HTML code. It's extensively used to stylize many web pages, including the page you're reading right now! What is the style attribute of HTML?

As I mentioned before, CSS provides a variety of properties for each tag in HTML. It gives an appeal for users to stay on our page. style is an inline tag that is essentially CSS, following the same property: value format.

What makes the the style attribute stand out from the standard CSS files, is that it can selectively change the design of any entity. If you wish to change the color of one tag, let's say <p>, to blue. Then, instead of a seperate file, you can directly add the property into the tag: 

<p style="color:blue">Hello There!</p>

Hello There!

As you can guess, the style attribute can be used to specify how a single tag looks. Unlike CSS where, unless specified, all the similar tags will go through the same formatting. That is, the same code above if written in a CSS file would make all paragraphs have the color blue.

Why can't we use HTML's style all the time?

You may ask yourself, why do we actually need a seperate CSS file? Why do we need a seperate file to do a job that can already be done in HTML?

While some requisites may be met by style, it still does not provide control over most of the visual structure of the web page. Of course, the previous two HTML examples were completely clean, no modifications were made to their looks. So let's say we do make some modifications, this time using the style attribute of HTML.

RANDOM HTML LINK

These are "buttons" made in HTML, styled with inline CSS properties. Clear duplicates of the CSS variants... well, almost. Thought it would be better to omit some features for this example. Why? Well, the problem with these buttons can be explained by their code, given below: 

 	<button style="background-color:#E0E; color:#FFF; border-radius:25px; border:0; font-family:Verdana; padding:15px;">HTML<br>DO NOTHING BUTTON</button>
	<a href='' style="background-color:#3B3; color:#FFF; border:0; padding:10px; text-decoration:none; border-radius:10px; font-family:Consolas; text-align:center; padding-top:20px;">RANDOM HTML LINK</a>

Multiple properties can be specified by adding a ; at the end. On topic, a code shouldn't be so long that one has to scroll right. Of course, you can format the above to look a bit more comprehensible, but there's a limit.

Such codes have great readability issues, making it more difficult to manage your code if you happen to format all of your tags like so. A fellow code editor may see this code, but might take some time to understand it. Another problem with this kind of formatting, is that you cannot use pseudo-selectors; like :hoverand:active.

There is another way to format tags, which is more readable and easier to understand. 

<style>
	button {
		background-color: #E0E; 
		color: #FFF;
		border-radius: 25px;
		border: 0;
		font-family: Verdana;
		padding: 10px;
	}
	
	a {
		background-color: #3B3;
		color: #FFF;
		border: 0;
		padding: 10px;
		text-decoration: none;
		border-radius: 10px;
		font-family: Consolas;
		text-align: center;
		padding-top:20px;
	}
	. . .
</style>

Here, this CSS block is embedded into HTML's head block using style. This would be how we write it in a CSS file as well. This way, we can add pseudo-selectors as we like.

But essentially, it would make your html file more long and crowded if you wish to give design to many elements. Not by a large margin, but may be enough to make one scrolling back and forth. These blocks are only suitable for formatting over some elements.

Should I always have a seperate CSS file?

Now, I made a lot of changes in that button and link. While it is of course a good practice to always a seperate file for your CSS properties, you should never have to create one that only changes, let's say, the color of your text, or your fonts. The <span> tag would be more suitable for giving such small designs.

Let's consider our previous example of giving blue color to a <p> tag. If we write it in a seperate file: 

 \* Kind of lonely here ;-; *\
	p {
		color: blue;
	}

Look at how lonely that <p> tag feels.

Remember, you can always use style inline for small visual changes. Shift to a style block inside HTML for bigger changes (like the code I will provide below). And if you feel you have added one too many, then perhaps it is time to create your new CSS file.
